# Promptsmith

**Promptsmith** is a TypeScript library for building type-safe, structured system prompts for AI agents using a fluent builder pattern.

## Why Promptsmith?

Creating effective system prompts for AI agents can be challenging:

- **Inconsistency**: Prompts vary in structure and completeness
- **Type Safety**: No validation of tool schemas and parameters
- **Maintainability**: Hard to update and version control scattered prompt strings
- **Integration**: Complex setup when integrating with AI frameworks

Promptsmith solves these problems by providing:

âœ… **Type-Safe Tool Definitions** - Define tools with Zod schemas for full type safety  
âœ… **Fluent Builder API** - Chain methods to build prompts incrementally  
âœ… **Structured Output** - Generate consistent, well-formatted markdown prompts  
âœ… **AI SDK Integration** - Seamless integration with Vercel AI SDK  
âœ… **Zero Dependencies** - Only requires Zod (which you likely already use)

## Quick Example

```typescript
import { createPromptBuilder } from "@promptsmith/core";
import { z } from "zod";

const prompt = createPromptBuilder()
  .identity("You are a helpful weather assistant")
  .capabilities([
    "Provide current weather information",
    "Give weather forecasts",
    "Explain weather patterns",
  ])
  .tool({
    name: "get_weather",
    description: "Get current weather for a location",
    schema: z.object({
      location: z.string().describe("City name or ZIP code"),
      units: z.enum(["celsius", "fahrenheit"]).optional(),
    }),
    execute: async ({ location }) => {
      // Your weather API call here
      return { temp: 72, condition: "sunny" };
    },
  })
  .constraint("must", "Always specify temperature units")
  .withTone("Be friendly and conversational")
  .build();

console.log(prompt);
```

This generates a structured markdown prompt ready to use with any AI model.

## Philosophy

Promptsmith follows a clear separation of concerns:

- **Prompt Generation** - What the AI model sees (text documentation)
- **Tool Execution** - What your application does (runtime logic)

Tools are registered with metadata for the AI to understand, while execution happens in your application when the AI requests to use a tool.
